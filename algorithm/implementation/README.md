# 구현(Implementation)

## 구현 알고리즘이란?

구현 알고리즘은 **머릿속에 있는 알고리즘을 정확하고 빠르게 프로그램으로 작성하는 것**을 의미한다.

코딩 테스트에서 구현이란 단순히 아이디어를 떠올리는 것이 아니라, 그 아이디어를 실제 소스코드로 옮겨 문제의 요구사항에 맞게 완성하는 과정이다.

즉,

> “머릿속의 알고리즘 → 실제 동작하는 코드”

로 바꾸는 능력이 바로 구현 역량이다.

#

## 구현 문제의 특징

보통 구현 문제는 다음과 같은 특징을 가진다.

- 풀이 아이디어 자체는 비교적 쉽게 떠오른다.
- 하지만 실제 코드로 작성하는 과정이 까다롭다.
- 조건이 많고 문제 설명이 긴 경우가 많다.
- 사소한 예외 처리 때문에 실수가 자주 발생한다.

> “문제 풀이 방법은 알겠는데 코드로 옮기기가 어렵다.”


<br>

## 구현 문제를 잘 풀기 위해 필요한 것

구현 문제에서는 알고리즘 지식만큼이나  
**프로그래밍 언어 문법 숙련도**가 중요하다.

특히 Kotlin에서는 다음 능력이 중요하다.

- 조건문/반복문 자유롭게 사용하기
- 문자열 처리
- 리스트 및 배열 조작
- 좌표 이동 구현
- 입력 처리
- 예외 상황 처리

> 문제의 요구사항을 실수 없이 코드로 작성하는 능력이 핵심이다.

<br>
<br>

## 구현 유형에 포함되는 대표 문제들

### 1. 완전 탐색 (Brute Force)

가능한 모든 경우의 수를 직접 계산하는 방식이다.

#### 예시
- 모든 좌표 이동 경우 확인
- 모든 숫자 조합 탐색
- 가능한 문자열 생성

#### Kotlin 예시

```kotlin
for (i in 1..3) {
    for (j in 1..3) {
        println("$i $j")
    }
}
```

#### 특징
- 단순하지만 구현량이 많아질 수 있다.
- 반복문과 재귀를 자주 사용한다.

#

### 2. 시뮬레이션 (Simulation)

문제에서 제시한 과정을 그대로 단계별로 수행하는 방식이다.

#### 예시
- 캐릭터 이동
- 게임 진행
- 로봇 움직임
- 시간 흐름 구현

#### Kotlin 예시

```kotlin
var x = 0
var y = 0

val dx = intArrayOf(-1, 1, 0, 0)
val dy = intArrayOf(0, 0, -1, 1)

for (i in 0 until 4) {
    x += dx[i]
    y += dy[i]
}
```

#### 특징
- 문제 조건을 정확히 해석해야 한다.
- 구현 실수가 자주 발생한다.

<br>

## 구현 문제에서 자주 어려운 부분

### 조건이 많은 문제

```kotlin
if (score >= 90) {
    println("A")
} else if (score >= 80) {
    println("B")
} else {
    println("C")
}
```

조건이 많아질수록 예외 처리 실수가 발생하기 쉽다.

#

### 문자열 파싱

문자열을 문자 단위로 처리해야 하는 경우가 많다.

```kotlin
val str = "Kotlin"

for (c in str) {
    println(c)
}
```

#

### 출력 형식 맞추기

구현 문제에서는 출력 형식을 정확히 맞춰야 한다.

예시:
- 공백 처리
- 줄바꿈
- 소수점 자리수 맞추기

```kotlin
val pi = 3.141592
println("%.2f".format(pi))
```

#

### 코드 길이 증가

알고리즘은 단순하지만 구현할 내용이 많아 코드가 길어지는 경우가 많다.

특히 아래 조건이 많아질수록 코드가 복잡해진다.
- 방향 처리
- 좌표 이동
- 상태 저장
- 예외 처리
