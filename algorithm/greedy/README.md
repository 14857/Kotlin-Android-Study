# Greedy 알고리즘

> 현재 상황에서 가장 좋아 보이는 선택을 하는 알고리즘

---

## Greedy 알고리즘이란?

Greedy(탐욕법) 알고리즘은 매 순간 가장 최선이라고 생각되는 선택을 하며 문제를 해결하는 방식이다.

- 현재 기준으로 가장 좋은 것을 선택하고
- 미래에 미칠 영향은 고려하지 않는다.



## 특징

### 1. 현재 최선의 선택

Greedy는 미래보다 현재를 우선한다.

예시:
- 가장 큰 값 선택
- 가장 작은 비용 선택
- 가장 빨리 끝나는 작업 선택

#

### 2. 정렬과 함께 자주 사용된다

다음과 같은 표현이 나오면 Greedy를 의심해볼 수 있다.

- 가장 큰 순서대로
- 가장 작은 순서대로
- 최소 비용
- 최대 개수

대부분 정렬을 활용해 해결한다.

#

### 3. 항상 정답이 되는 것은 아니다

Greedy는 매우 직관적이지만 모든 문제에서 최적해를 보장하지는 않는다.

따라서:

> "왜 이 방법이 항상 정답인가?"

를 설명할 수 있어야 한다. 이를 Greedy 알고리즘의 정당성이라고 한다.  


<br>

## 거스름돈 문제

### 문제

1260원을 최소 개수의 동전으로 거슬러 주기

사용 가능한 동전:
- 500원
- 100원
- 50원
- 10원

#

### 해결 아이디어

가장 큰 단위의 동전부터 최대한 사용한다.

#

### Kotlin 코드

```kotlin
fun main() {

    var money = 1260
    var count = 0

    val coins = listOf(500, 100, 50, 10)

    for (coin in coins) {

        count += money / coin
        money %= coin
    }

    println("동전 개수: $count")
}
```

#

### 실행 결과

```text
동전 개수: 6
```

#

### 시간 복잡도

동전 종류 수를 K라고 하면:

```text
O(K)
```

동전 종류만큼 반복하므로 매우 빠르다.

#

### Greedy 알고리즘의 정당성

이 문제에서 Greedy가 가능한 이유는 큰 단위의 동전이 항상 **작은 단위의 배수**이기 때문이다.

- 500 = 100 × 5
- 100 = 50 × 2
- 50 = 10 × 5

따라서 큰 동전을 먼저 사용하는 것이 항상 최적해가 된다.

#

### 자주 나오는 Greedy 유형

| 문제 유형 | 핵심 아이디어 |
|---|---|
| 거스름돈 | 큰 단위부터 사용 |
| 회의실 배정 | 빨리 끝나는 회의 선택 |
| 큰 수 만들기 | 현재 가장 큰 숫자 선택 |
| 카드 정렬 | 가장 작은 묶음부터 합치기 |
| 체육복 | 가까운 학생에게 빌려주기 |
